# Bank reconciliation

Match imported bank statement lines to JE items on the bank account.
Statement is a Mongoose doc with embedded `lines[]`; each line can be
matched to a JE item via the ledger's reconciliations service.

```
POST /accounting/bank-reconciliation { bankAccountCode, statementDate, lines: [...] }   resource CRUD
  → BankStatement.create()                  bank-statement.model.ts
  ← returns the statement doc (status: 'draft')

GET /accounting/bank-reconciliation/open-items?bankAccountCode=1111      bank-reconciliation.resource.ts
  → Account.findOne({ accountNumber: bankAccountCode })
  → accounting.repositories.reconciliations.getOpenItems({ accountId, organizationId, asOfDate? })
  ← unmatched JE items on that bank account

POST /accounting/bank-reconciliation/:id/action {action: "matchLine", ...}
  → validate lineIndex + jeEntryId + jeItemIndex + jeAccountId
  → BankStatement.findById to read the line — must NOT already be matched
  → accounting.repositories.reconciliations.match({ account, items: [...], organizationId })
  → updates BankStatement.lines[i] with { matchingNumber, jeEntryId, jeItemIndex }
  ← returns { matchingNumber }

POST /accounting/bank-reconciliation/:id/action {action: "unmatchLine", lineIndex}
  → line must already be matched
  → accounting.repositories.reconciliations.unmatch({ matchingNumber, organizationId })
  → clears matchingNumber/jeEntryId/jeItemIndex on the line
  ← returns { unmatched: true }
```

## Gotchas

- **`bankAccountCode` is the CoA account NUMBER**, not the ObjectId. The
  matchLine handler accepts either: tries `Account.findOne({ accountNumber: jeAccountId })`
  first, then falls back to interpreting `jeAccountId` as an ObjectId.
- **Statement import is NOT wired** — the resource has CRUD but no
  CSV/MT940 parser. Statements must be POSTed with `lines[]` already
  shaped by the caller. Likely target for future import worker.
- **`matchingNumber` is opaque** — generated by the ledger reconciliations
  service. Don't construct it manually; always pass through `match()`.
